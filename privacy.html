Privacy Policy

Last updated: May 2025

Gym Tracker is a personal fitness tracking application. This policy describes how the application handles data, including data obtained through the WHOOP API.

1. Personal Use Only

This application is operated solely for personal use by its developer. It is not a commercial product and has no registered users other than the developer.

2. Data Collected

The application may access the following data from WHOOP, solely with the explicit authorization of the account owner:

  • Recovery score, HRV, resting heart rate, and SpO2
  • Sleep performance and duration
  • Workout strain, heart rate, and activity data
  • Daily cycle data

3. How Data Is Stored

All data — including WHOOP health metrics and workout logs — is stored exclusively in the user's own browser (localStorage / IndexedDB) and optionally in a private Firebase Firestore database accessible only by the account owner. No data is stored on any third-party server controlled by this application.

4. Data Sharing

No data is shared with any third party. No analytics, advertising, or tracking services are used. No data is sold, rented, or disclosed to any external entity.

5. WHOOP API Access

WHOOP credentials (OAuth tokens) are stored locally in the user's browser and are used exclusively to retrieve the account owner's own health data. Tokens are never transmitted to any server other than WHOOP's official API endpoints (api.prod.whoop.com).

6. Data Retention

Data persists in the browser until the user manually clears it or revokes WHOOP API access. The user can disconnect WHOOP at any time from within the application settings.

7. Contact

For any questions about this policy, contact the developer at: github.com/aturpin88/gym-tracker
